# US-041: Investment TypeScript Types

## Story
As the Insight platform user, I want strongly typed data models for the investment domain so that all services and calculations have a reliable, self-documenting contract to work against.

## Dependencies
- US-001: Project Scaffolding

## Requirements
- Add the following TypeScript types to `src/types/index.ts` (alongside any existing types):

**Portfolio** (PRD §6.1.1):
- `id`: string (PocketBase auto-generated)
- `name`: string (e.g. "My Portfolio", "Erik's Portfolio")
- `ownerName`: string (who the portfolio is for, e.g. "Me", "Erik")
- `isDefault`: boolean (exactly one per user is default)
- `owner`: string (FK to users collection — data isolation)
- `created`: string (ISO datetime)

**Platform** (PRD §6.1.2):
- `id`: string
- `portfolioId`: string (FK to Portfolio)
- `name`: string (e.g. "Nordnet", "Interactive Brokers")
- `icon`: string (image file reference via PocketBase)
- `type`: `"investment" | "cash"` (determines UI treatment and available metrics)
- `currency`: string (e.g. "DKK", "EUR" — drives display formatting and conversion)
- `status`: `"active" | "closed"` (closed platforms muted, excluded from current totals)
- `closedDate`: string | null (optional, ISO date — date the platform was closed)
- `closureNote`: string | null (optional — reason/context for closing)
- `owner`: string
- `created`: string

**DataPoint** (PRD §6.1.3):
- `id`: string
- `platformId`: string (FK to Platform)
- `value`: number (total platform value in native currency)
- `timestamp`: string (ISO datetime, defaults to now)
- `isInterpolated`: boolean (default false; true if generated by month-end interpolation)
- `note`: string | null (optional)
- `owner`: string
- `created`: string

**Transaction** (PRD §6.1.4):
- `id`: string
- `platformId`: string (FK to Platform)
- `type`: `"deposit" | "withdrawal"`
- `amount`: number (always positive; sign derived from type)
- `exchangeRate`: number | null (optional — rate at time of transaction for non-DKK platforms)
- `timestamp`: string (ISO datetime)
- `note`: string | null (optional)
- `attachment`: string | null (optional — PocketBase file field reference)
- `owner`: string
- `created`: string

**ExchangeRate** (PRD §4.1.1):
- `id`: string
- `fromCurrency`: string (e.g. "EUR")
- `toCurrency`: string (always "DKK")
- `rate`: number (e.g. 7.46 means 1 EUR = 7.46 DKK)
- `date`: string (ISO date — the date this rate applies to)
- `source`: `"auto" | "manual"` (auto = fetched from API, manual = user override)
- `owner`: string
- `created`: string

**Settings** (PRD §3.8, §10):
- `id`: string
- `userId`: string (relation to users)
- `dateFormat`: `"YYYY-MM-DD" | "DD/MM/YYYY"` (default: "YYYY-MM-DD")
- `theme`: `"light" | "dark"` (default: "light")
- `demoMode`: boolean (default: false)
- `created`: string

- Define union literal types for reuse:
  - `PlatformType = "investment" | "cash"`
  - `PlatformStatus = "active" | "closed"`
  - `TransactionType = "deposit" | "withdrawal"`
  - `ExchangeRateSource = "auto" | "manual"`
  - `DateFormat = "YYYY-MM-DD" | "DD/MM/YYYY"`
  - `Theme = "light" | "dark"`
  - `Currency = "DKK" | "EUR" | string` (extensible for future currencies)

- Define input/creation types (omitting id, created, owner) for each entity:
  - `PortfolioCreate`: Omit<Portfolio, "id" | "created" | "owner">
  - `PlatformCreate`: Omit<Platform, "id" | "created" | "owner" | "closedDate" | "closureNote"> with closedDate and closureNote optional
  - `DataPointCreate`: Omit<DataPoint, "id" | "created" | "owner">
  - `TransactionCreate`: Omit<Transaction, "id" | "created" | "owner">
  - `ExchangeRateCreate`: Omit<ExchangeRate, "id" | "created" | "owner">

## Technical Notes
- File to modify: `src/types/index.ts` — add investment types alongside any existing types
- Use `type` (not `interface`) for consistency with union types
- All optional fields use `| null` rather than `?` to make nullability explicit in API responses
- PocketBase returns ISO string dates, so all date/datetime fields are typed as `string`
- The `owner` field on every type enables future multi-user support and ensures data isolation via PocketBase filter rules
- `Currency` is typed as a union with string fallback (`"DKK" | "EUR" | string`) so new currencies can be added without type changes per PRD §4.4
- The `icon` field on Platform is a PocketBase file field (image upload) per PRD §9.2 and §10
